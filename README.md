API Integration Web Application

#Project Overview

This project demonstrates the implementation of API integration in web development by creating a responsive web application that dynamically fetches and displays data from a public API, such as a Weather API or News API. The application is built using core web technologies and focuses on retrieving real-time data from an external source and presenting it in a structured and user-friendly format.

The primary goal of this project is to understand how frontend applications interact with external APIs without using any backend frameworks. The project highlights the use of client-side scripting to handle asynchronous data fetching and dynamic content rendering. This approach reflects how many modern web applications deliver real-time information to users.

#Task Description

The task involved developing a responsive webpage that integrates with a public API to fetch real-time data dynamically. The webpage sends HTTP requests to the API, receives data in JSON format, processes the response, and displays relevant information on the webpage.

The application is designed to be lightweight and easy to run, requiring no additional libraries or frameworks. The webpage can be executed directly in a browser, making it accessible and simple to use. The task also helped in understanding the fundamentals of asynchronous programming, error handling, and dynamic UI updates using JavaScript.

#Tools and Technologies Used Technologies

HTML5 Used to define the structure and layout of the webpage, including input fields, buttons, and content containers.

CSS3 Used to style the webpage and ensure responsiveness across different screen sizes and devices.

JavaScript (ES6) Used to implement API integration, handle asynchronous requests using the Fetch API, parse JSON responses, and dynamically update the webpage.

Public API (Weather API / News API) Used to retrieve real-time data from an external source.

#Editor and Execution Platform

Notepad Used as the primary text editor for writing HTML, CSS, and JavaScript code. This demonstrates a strong understanding of core web technologies without reliance on advanced development environments.

Command Prompt (Windows) Used to navigate project directories, manage files, and run the application where required. The project does not require a backend server and runs directly in a web browser.

Web Browser (Google Chrome) Used to execute and test the application and verify the dynamic loading of API data.
